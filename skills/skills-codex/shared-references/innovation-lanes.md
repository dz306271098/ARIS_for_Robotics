# Innovation Lanes

Use these lanes whenever a workflow needs deliberate creative expansion before convergence.

The objective is to prevent ideation and planning from collapsing too early into a single safe route.

## Default Lanes

### 1. Gap-Closing Lane

Start from explicit literature gaps.

Questions:
- Which bottleneck is repeated across multiple papers?
- Which missing evidence package would materially change the field?
- Which capability is clearly wanted but still unsupported?

### 2. Cross-Domain Analogy Lane

Import a principle from another field with the same structural problem.

Questions:
- What other field has deeper theory for this sub-problem?
- Which mechanism solved the analogous observability / uncertainty / coordination / search issue there?
- What is the smallest valid transfer into this domain?

### 3. Contradiction-Resolution Lane

Turn conflicting literature or empirical findings into a research target.

Questions:
- Which two papers seem to imply incompatible conclusions?
- Is the contradiction due to data regime, benchmark bias, hidden assumption, or evaluation mismatch?
- Could resolving the contradiction itself become the contribution?

### 4. Anti-Assumption Lane

Attack an assumption the field keeps making.

Questions:
- What does everyone treat as necessary but rarely test?
- What privileged signal, scaling law, or training heuristic is being taken for granted?
- What if the assumption is false or only conditionally true?

### 5. Failure-Reframing Lane

Mine failed ideas and negative results for new routes.

Questions:
- What did the failure actually teach us about the hidden structure of the problem?
- Under what narrower conditions could the failed route revive?
- What principle from the failed attempt is still valid even if the method was not?

## Portfolio Rule

Every ideation or early planning stage should preserve at least three route types unless the evidence is overwhelming:

- `safe` — strongest evidence-backed route
- `bold` — highest upside route with credible mechanism
- `contrarian` — route that challenges a dominant assumption or common framing

If two lanes generate the same idea family, merge them. Otherwise keep them separate long enough to survive novelty, review, and cheap pilot pressure.

## Convergence Rule

Do not collapse the portfolio to one route until:
- novelty risk is mapped
- at least one reviewer pass has attacked the portfolio
- cheap pilot or decisive evidence has killed the weaker branches

## Logging Rule

For every retained route, record:
- source lane
- core principle(s)
- closest prior work
- main kill criterion
- first decisive experiment
