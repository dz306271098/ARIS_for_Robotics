# Literature Synthesis Protocol

Use this protocol when turning a large paper set into innovation-ready research assets.

The goal is not a long survey. The goal is a structured synthesis that supports ideation, route design, failure recovery, and claim control.

## Required Outputs

### 1. Landscape Map

Group the literature by:
- approach family
- problem framing
- hidden assumption
- failure mode
- evaluation regime

### 2. Principle Bank

For the most relevant papers, apply `principle-extraction.md` and keep only the principles that are transferable.

### 3. Analogy Candidates

For cross-domain or adjacent-field papers, record:
- source field
- shared structural problem
- transferable principle
- expected upside
- adaptation risk

### 4. Novelty Surface

Map novelty risk at the claim level:
- exact duplicates
- same mechanism in another domain
- same setting with different mechanism
- likely reviewer citations even if not exact matches

### 5. Failure and Saturation Signals

Record:
- recurring negative results
- benchmark saturation warnings
- evaluation traps
- common hidden infrastructure that inflates results

## Compression Rule

When context budget is tight, compress in this order:
- keep failed ideas and failure signals
- keep top principles
- keep top analogy candidates
- keep novelty surface
- compress prose summaries first

Never compress away anti-repetition memory before removing descriptive fluff.

## Handoff Rule

A literature stage is not complete until downstream stages can answer these questions without rereading the full paper set:

- What are the top unresolved bottlenecks?
- Which principles look transferable?
- Which assumptions should be challenged?
- Which ideas are already dead ends?
- Which claims are most likely to fail novelty or reviewer scrutiny?
